# Координация — кто что трогает

Читать перед стартом сессии. Обновлять, когда начинаешь/заканчиваешь работу над зоной.
Цель: не редактировать один файл вслепую вдвоём одновременно.

## Правила

1. **Перед стартом:** `git pull origin main`, прочитать этот файл.
2. **Занял зону** — добавь строку в таблицу ниже (кто, что, с какого времени).
3. **Закончил** — убери свою строку, запушь.
4. **Один файл — один человек одновременно.** Если зона занята — бери другую или спроси в чате.
5. Коммитить часто, мелкими порциями. Не копить правки на весь день молча.
6. PR в `main` — не пушить напрямую в обход ревью на спорных/крупных изменениях (для мелких правок прямой push в main ок, как сейчас).

## Кто что сейчас трогает

| Кто | Зона / файлы | С какого времени | Заметка |
|---|---|---|---|
| _пример_ | `/sistema/`, `/sayty/` | 2026-07-12 14:00 | марафон 10/10 |

## Зоны ответственности (по умолчанию, если не указано иначе)

- Матвей: маркетинговые страницы, дизайн, копирайт
- Партнёр: _вписать_

## Полезные команды

```bash
git pull origin main              # синхронизация перед стартом
git add <конкретные файлы>        # не git add -A, чтобы не сгрести чужие правки
git commit -m "..."
git push origin main
```

Если словили конфликт при `git pull` — НЕ `git checkout .` / `git reset --hard` не разобравшись. Сначала `git status`, посмотреть что конфликтует, решить руками или спросить.
